# 기본 설계 구조

## 계층

### App

- 애플리케이션의 전반적인 구성을 관리하는 역할
- 전반적인 설정을 구성하고, 컨트롤러 등록 등을 담당

### Controller

- 애플리케이션에 전달된 명령을 분기하고 처리하는 역할
- 처리 가능한 요청들을 정의하고 해당 요청을 서비스로 전달

### Service

- 컨트롤러에서 전달된 요청에 대해 실제 비즈니스 로직을 수행하는 역할
- 서비스 단위로 필요한 작업들을 정의하고, 엔티티를 조작하여 리포지토리와 상호작용

### Repository

- 데이터베이스 혹은 외부 시스팀과의 실제 상호작용을 담당하는 역할
- 조작된 엔티티를 저장하거나, 조회된 결과를 엔티티 형태로 변환하여 반환

### Persistence

- 데이터베이스 혹은 외부 시스템과의 연결을 설정하는 역할
- 실제로 수행해야 하는 동작을 기준으로 라이브러리 메서드를 호출

## 구성요소

### Entity

- 도메인 단위로 데이터베이스 혹은 외부 시스템의 정보를 표현하는 객체
- 실제 비지니스 로직에서 조작되는 주요 객체

### Value Object

- 특정 값을 표현하거나, 유효성을 판별하는 등, 세부 정책이 관리되는 객체
- 도메인 내에서 특정 값의 유효성 검사 및 조작을 담당

---

## 작업 순서

1. 앱 계층에 전체 프로그램 구성을 정리
2. 컨트롤러 계층에 처리할 작업들을 정리
3. 데이터베이스 스키마 정의
4. 정리된 스키마 기준으로 도메인 정의
5. 도메인 별로 스키마에 맞는 엔티티 준비
6. 도메인 별로 서비스 계층을 추가하여 비즈니스 로직 구현
7. 레포지토리 계층에서 엔티티 조작 구현
8. 비지니스 로직의 유효성 검사 및 값 조작 동작을 값 객체로 분리
